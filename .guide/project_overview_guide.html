📖 Project Overview & Roadmap

Your guide to understanding iMatchBook's architecture and progress

🎯 What is iMatchBook?

iMatchBook is an intelligent expense reconciliation platform. It allows users to import bank transaction data and use AI to automatically match those transactions with photos of physical receipts.

🔧 Backend Deep Dive

The backend is built with PHP and acts as a stateless API bridge.

Key Components:

  • api/import_bank.php: Uses dynamic header detection. It doesn't rely on fixed column orders, allowing it to work with multiple banks.
  • api/upload.php: Communicates with Google Cloud using a Service Account key (.json). It handles binary image uploads and interprets Document AI's JSON response into a normalized format.
  • config/db.php: Centralized connection logic using PDO for security.

🌍 Real Server Deployment Tips

If you are moving this to a production VPS (like DigitalOcean, AWS, or a dedicated server):

  1. SSL is Mandatory: Camera capture (getUserMedia) requires HTTPS in modern browsers.
  2. Environment Keys: Ensure your .json service keys for Google Cloud are placed outside the public web root or protected by .htaccess.
  3. WAF / CORS: Update Access-Control-Allow-Origin in the PHP files to point to your domain instead of *.
  4. Docker Production: Use docker-compose -f docker-compose.yml up -d but ensure you have a reverse proxy (Nginx/Traefik) handling SSL.
